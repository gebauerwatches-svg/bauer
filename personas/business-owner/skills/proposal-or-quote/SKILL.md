# Proposals and quotes

When the user asks for a proposal or quote:

## Step 1 — Get the basics

Ask in one question (skip what's clear):
- **What's the work?** (One sentence — what you're delivering.)
- **For whom?** (Company name, contact name if mentioned.)
- **Price range?** (Approximate; or hourly rate + estimated hours.)
- **Timeline?** (Start date, end date, milestones.)

## Step 2 — Output a structured draft

```
[Project name]
For: [client]

What's included:
- [deliverable 1]
- [deliverable 2]
- [deliverable 3]

Timeline: [start] → [end]
Investment: [price]

What's NOT included: (so there's no scope confusion later)
- [out-of-scope item 1]
- [out-of-scope item 2]

Payment terms: [e.g., 50% upfront, 50% on delivery]

Next step: [e.g., reply to confirm and we'll send a contract]
```

## Rules

- **Always include "What's NOT included."** Saves the user a scope fight later.
- **Lead with deliverables, not process.** Clients buy outcomes.
- **Round prices** when communicating externally.
- **Note assumptions** the price depends on (e.g., "assumes 2 rounds of revisions").

## After drafting

End with: "Want me to make this more formal, more casual, or split into a longer document?"

For anything legally binding (contracts, NDAs, warranties), add at the end:

> *Note: this is a working draft. Have a lawyer review before sending.*
