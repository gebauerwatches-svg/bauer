# Drafting content

When the user asks for help drafting content:

## Step 1 — Get the basics

Ask in one question (skip if clear):
- **What format?** (Newsletter, blog, Twitter thread, LinkedIn post, etc.)
- **What's the angle / point?** (One sentence — what's the takeaway for the reader.)
- **Who's the reader?** (Skip if obvious from the user's prior context.)

## Step 2 — Draft

Match the format:

- **Newsletter post:** 300-800 words. Personal voice. Hook → story → takeaway → CTA (often "hit reply and tell me X").
- **Blog post:** 600-1500 words. Subheads. Scannable. End with the reader's next step.
- **Twitter thread:** 5-12 tweets, each one stands alone but they hang together. First tweet earns the click. Last tweet has the takeaway or the CTA.
- **LinkedIn post:** ~150-300 words. Conversational. Whitespace matters. End with a question.
- **Podcast outline:** intro hook (30 sec) → 3-5 main points with examples → CTA (subscribe / next episode tease).

## Rules

- **Earn the read.** First sentence has to be a reason to keep reading. No "Today I want to talk about..."
- **One main idea per piece.** Don't try to teach 4 things in one post.
- **Specific over abstract.** Real numbers, real examples, real names.
- **Match the user's voice** if you've seen samples. Otherwise: direct + warm.
- **No buzzwords or guru-speak.**

## Length

- Default to the SHORT end of typical for that format. Cut, don't expand.
- If the user's idea fits in 200 words, deliver 200 words. Don't pad.

## End with

"Want me to make it shorter, change the angle, or write the headline differently?"
