About

Hi, I’m Liam.

14 years old. Steamboat Springs, Colorado. I built Bauer.

Why I built this

I’d been using Claude Code for a while when I started noticing something: a lot of people in my life kept asking if I could help them set it up. They’d heard about it, they wanted to use it — but the install and config was confusing, and most of them weren’t developers.

They didn’t want to build software. They wanted help with normal stuff — running their small business, working on a side project, writing better, getting unstuck on whatever they were doing.

The problem was that all the docs and examples and YouTube videos out there were written for developers. The default install drops you into a blank assistant with no CLAUDE.md, no skills, and no idea of what you actually do. So you either spend hours figuring out how to configure it, or you give up.

That’s why I built Bauer. It’s a one-command installer that does the messy parts for you and drops in a config that’s tuned for what you actually do — running a business or shipping a side project — instead of for writing software.

Why “Bauer”

“Bauer” is German for builder. It’s also a piece of my last name — Gebauer, which is the older form of the same word. My family name comes from people who built and grew things for a living, going back generations in Germany and Austria.

It felt like the right name for a tool whose whole job is to help people build and grow their own work — whether that’s a small business, a side project, a newsletter, a company, or whatever they came here to do.

What I want this to be

I want Bauer to be the easiest way for non-developers to use Claude Code. If you’re running a small business, working on a side project, vibe coding something, or building a company — you should be able to install AI help in 60 seconds and have it tuned for what you actually do, not the developer defaults.

If this helps even a few people skip the setup grind and get to the actual work, that’s a win for me.

What’s next

Bauer is in v0.1. I’m working on:

  • Adding more personas (freelancers and consultants are next)
  • Better install error handling for tricky setups
  • An easy way to update your config as your work evolves
  • Walkthroughs and tutorials for people brand-new to AI
